```markdown
- 문제 1: [문제 설명]
```

## 일지 예시 - 1일차

```markdown
# 개발 일지 - 1주차 (일 1)

## 오늘의 목표
- Unity 프로젝트 초기 설정
- 기본 폴더 구조 생성
- Git 저장소 설정
- 프로젝트 네임스페이스 정의

## 완료한 작업
- Unity 2022.3 LTS 프로젝트 생성 완료
- 기본 폴더 구조 설정 (Scripts, Prefabs, ScriptableObjects, Scenes 등)
- Git 저장소 초기화 및 .gitignore 설정
- 어셈블리 정의 파일 생성 및 네임스페이스 구조 설계

## 문제점 및 해결책
- 문제 1: Unity 패키지 매니저에서 MagicOnion 종속성 해결 이슈
  - 해결책: UPM Git URL을 사용하여 직접 패키지 참조
- 문제 2: 어셈블리 정의 파일 작성 시 참조 순환 발생
  - 해결책: 공통 코드를 Core 어셈블리로 분리하여 참조 구조 개선

## 학습 사항
- Unity 어셈블리 정의 시스템 작동 방식 학습
- MagicOnion 통합을 위한 사전 요구사항 확인
- 최신 Unity 버전에서의 폴더 구조 모범 사례 검토

## 내일 계획
- 핵심 관리자 클래스 설계 및 구현
- 기본 데이터 구조 정의
- ScriptableObject 설계
- 시스템 간 상호작용 설계

## 메모
- MagicOnion 문서 검토 결과, gRPC 종속성 버전 충돌 가능성이 있음. 주의 필요.
- 타입 안전성과 유지보수성을 위해 ScriptableObject를 최대한 활용하는 것이 좋을 것으로 판단됨.
- 던전 생성 알고리즘 구현 시 메모리 사용량과 생성 속도 균형 고려 필요.
```

## 일지 예시 - 3일차

```markdown
# 개발 일지 - 1주차 (일 3)

## 오늘의 목표
- MagicOnion 및 필수 패키지 설치
- MessagePack 설정 및 리졸버 구성
- 기본 네트워크 서비스 및 허브 인터페이스 정의
- 간단한 통신 테스트

## 완료한 작업
- MagicOnion v4.5.0 및 MessagePack v2.5.108 설치 완료
- MessagePack 리졸버 및 포맷터 구성 완료
- IGameService 및 IGameHub 인터페이스 정의
- 동기화를 위한 기본 데이터 구조 정의
- 간단한 에코 테스트 통신 구현 및 성공

## 문제점 및 해결책
- 문제 1: MessagePack 초기화 시점 오류
  - 해결책: RuntimeInitializeOnLoadMethod 속성을 사용하여 리졸버 초기화 시점 조정
- 문제 2: Unity IL2CPP 빌드 시 AOT 이슈
  - 해결책: MessagePackInitializer 클래스에 AOT 코드 생성 지원 추가
- 문제 3: MagicOnion 서버-클라이언트 연결 중 DNS 해결 이슈
  - 해결책: IP 주소를 직접 사용하고 추후 DNS 해결 문제 조사 예정

## 학습 사항
- MagicOnion의 허브와 서비스 개념 차이점 이해
  - 서비스: 단방향 RPC 호출 (HTTP와 유사)
  - 허브: 양방향 실시간 통신 (SignalR과 유사)
- MessagePack 최적화 및 형식 유지 방법
- gRPC 통신 디버깅 기법

## 내일 계획
- 네트워크 매니저 클래스 구현
- 서버 연결 및 세션 관리 코드 작성
- 오류 처리 및 재연결 메커니즘 구현
- 기본 서버 프로젝트 설정

## 메모
- 로컬 개발 환경에서는 성능이 좋지만, 네트워크 지연 시뮬레이션 테스트 필요
- 서버-클라이언트 간 데이터 동기화를 위한 전략 추가 검토 필요
- MessagePack 직렬화 최적화 방안 추가 조사
```

## 일지 예시 - 5일차

```markdown
# 개발 일지 - 1주차 (일 5)

## 오늘의 목표
- 던전 관련 데이터 구조 정의
- 던전 생성 매개변수 및 구성 클래스 구현
- BSP 노드 클래스 및 기본 구조 구현
- 초기 공간 분할 알고리즘 구현

## 완료한 작업
- Room, Corridor, Dungeon 등 기본 데이터 클래스 구현
- ScriptableObject 기반 던전 생성 설정 클래스 구현
- BSP 트리 노드 클래스 및 분할 알고리즘 구현
- 기본 공간 분할 및 리프 노드 추출 로직 구현
- 간단한 디버그 시각화 도구 구현

## 문제점 및 해결책
- 문제 1: BSP 알고리즘에서 불균형한 분할 발생
  - 해결책: 분할 비율에 임의성을 추가하고 최소/최대 비율 제한 설정
- 문제 2: 재귀적 분할 시 스택 오버플로 위험
  - 해결책: 최대 깊이 제한 및 분할 조건 추가 검증
- 문제 3: 방 생성 시 겹침 현상 발생
  - 해결책: 방 크기에 여백(패딩) 적용 및 최소 방 크기 설정

## 학습 사항
- BSP 알고리즘 최적화 방법 및 변형
- Unity에서 2D 그리드 데이터 효율적 관리 방법
- 프로시저럴 콘텐츠 생성(PCG)의 기본 원리

## 내일 계획
- 방 생성 및 배치 로직 구현
- 복도 연결 알고리즘 구현
- 시각화 도구 개선
- 다양한 매개변수로 알고리즘 테스트

## 메모
- BSP 알고리즘은 균형 잡힌 방 배치에 효과적이지만, 자연스러운 던전 형태를 위해 추가 변형 필요
- 특수 방(입구, 출구, 보물 등) 배치 규칙 고려 필요
- 추후 다른 알고리즘(Cellular Automata, Drunk Walk 등)과 조합하는 방안 검토
```

```markdown
# 1주차 회고: 기반 시스템 구축

## 계획 대비 진행 상황
- [x] Unity 프로젝트 설정 및 기본 구조 구축: 100% 완료
- [x] MagicOnion 통합 및 기본 네트워크 구조: 90% 완료

- [ ] 시스템 통합 및 최적화: 70% 완료

- Unity 프로젝트와 기본 아키텍처를 성공적으로 설정
- BSP 알고리즘 기반 던전 생성기 구현 및 시각화
```
